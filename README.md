## Step 2 – Explore & Summarize Project Management Processes

This step focused on reviewing and understanding the project management processes used in a GitHub-based development workflow and how GitHub Copilot supports those processes.

### Project Management Process Overview

The project management workflow follows standard GitHub practices:

- Work is defined and tracked using GitHub Issues.
- Development tasks are implemented in feature branches.
- Changes are reviewed and validated through Pull Requests.
- Documentation is updated to reflect decisions and outcomes.

### Role of GitHub Copilot

GitHub Copilot supports the project management lifecycle in multiple ways:

- **Copilot Chat** assists with:
  - Understanding repository structure and existing code
  - Clarifying requirements and implementation approaches
  - Drafting and improving documentation such as README files
  - Summarizing changes and discussions

- **Copilot Coding Agent** (where enabled) can:
  - Work autonomously on assigned GitHub issues
  - Propose solutions through draft pull requests
  - Iterate based on reviewer feedback

> Note: Access to the Copilot Coding Agent is controlled by subscription level and organizational policy. In this environment, the agent was not enabled, so exploration focused on conceptual understanding rather than hands-on execution.

### Outcome

This step provided an understanding of how GitHub Copilot integrates into modern project management workflows, emphasizing collaboration, human oversight, and responsible use of AI-assisted development tools.
